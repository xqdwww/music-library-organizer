# Contributing

Small fixes, tests, and documentation improvements are welcome. Please open an issue before proposing a new format, metadata source, or workflow so the project can remain low-maintenance and keep all media operations local.

Contributions must not add media downloading, personal-library authentication, private APIs, general-purpose scraping, DRM circumvention, remote storage management, real music fixtures, or user data. A public metadata adapter requires documented terms, strict response bounds, caching, rate limits, privacy documentation, fixture-only tests, and explicit maintainer review. Run the unit tests and `ruff check .` before submitting a pull request. By contributing, you agree that your contribution is licensed under GPL-2.0-or-later.
